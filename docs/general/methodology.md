# DataReady Consulting Methodology

## What We Do
DataReady helps organisations assess their data and AI readiness, identify capability gaps, and build actionable roadmaps. We work across data governance, analytics strategy, and AI implementation.

## Data Readiness Assessment Framework
A data readiness assessment evaluates five maturity dimensions:
1. **Data Quality** — completeness, accuracy, consistency, timeliness of data assets
2. **Governance** — policies, ownership, lineage, cataloguing, and compliance controls
3. **Infrastructure** — data platform architecture, pipelines, storage, and tooling
4. **People & Process** — team capability, data literacy, and operational workflows
5. **Analytics & AI** — current use of analytics, ML maturity, and AI adoption

Scores are given 1–10 per dimension. Overall readiness is classified as: Critical (1-2), Low (3-4), Developing (5-6), Moderate (7-8), Strong (9-10).

## DAMA-DMBOK Framework
DataReady applies DAMA-DMBOK (Data Management Body of Knowledge) as the foundation for governance assessments. Key knowledge areas include: Data Governance, Data Architecture, Data Modelling, Data Storage, Data Security, Data Integration, Document & Content Management, Reference & Master Data, Data Warehousing & BI, Metadata Management, and Data Quality.

## Discovery Engagement Process
1. **Intake** — client submits problem statement and focus areas
2. **Assessment** — automated readiness scoring + gap identification
3. **Briefing** — structured consulting brief with next steps and governance flags
4. **Follow-up** — conversational refinement and 90-day action planning
5. **Delivery** — prioritised roadmap with owner, timeframe, and effort per action

## Common Quick Wins
- Document a data dictionary for the top 5 most-used datasets
- Assign data owners to critical data domains
- Run a data quality profiling exercise on priority data sources
- Establish a weekly data issue triage meeting
- Create a single source of truth for key business metrics

## Key Questions in Discovery
- Who currently owns data quality in the organisation?
- Is there a data catalogue or metadata management system in place?
- What are the top 3 data-driven decisions made each month?
- Where does data currently live and how does it move between systems?
- What regulations apply to the data you collect and process?
